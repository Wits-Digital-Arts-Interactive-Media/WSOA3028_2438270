Desktop Wireframes Version 1

by Brad Simbizi

Iteration 1 Wireframes for the desktop website:
Here are my 1st iteration wireframes for the various pages I'll have on the website. I have tried to list some advantages and disadvantages for each layout.

Desktop Home page idea 1

The main reason for this design choice, was its simplicity and effectiveness. I feel like this website design would accomplish its intended purpose. Viewers would be able to navigate the website with relative ease. There is no unnecessary clutter to distract or overwhelm the viewer. This seems to be a positive and negative. The viewer isn’t distracted by the backgrounds or a complicated UI layout. However, the simplistic design might be way too boring for some viewers.

Desktop Home page idea 2:

The framework of this design and the last are effectively the same. The main aim was for a clean, minimalist appearance. A non-cluttered UI interface. Similar to the first design, the main issue with this layout is it effectively boring to look at. It feels very cookie-cutter. Which isn’t a bad thing, but it really soaks up excitement like a sponge.

Home page idea 3:

This is the design I liked out of the 3. This design is also very simplistic. However, the floating lettering telling the viewer what page they are on, as well as the uniform square buttons provide a much-needed touch of excitement. A cool little logo would also push the excitement factor further. The only thing I’m left questioning is the home button. It seems kind of useless to have a home button on the home page.

Home page idea 4:

This layout removes the home button from the home page.

Blog page idea 1:

This layout is an extension of Home page idea 4. For this layout, it makes more sense to have the home button in the top center of the page. The floating lettering remain to give the page a cooler appearance. With this layout, the viewer can see all the blog posts as they load into the page. My only gripe with this is that it looks quite cluttered.

Blog page idea 2:

This layout removes the cluttered look of all the blog post buttons, replacing them with larger buttons in a row. For this layout, the viewer would have to scroll across to access the other blog posts.

Blog post idea 3:

This layout simply replaces the scrollbar with a button.

Portfolio page idea 1:

Like the blog post page, this variation employs a scrollbar.

Portfolio page idea 2:

This layout uses buttons for navigation.

Essay page idea 1:

This layout uses a scrollbar.

Essay page idea 2:

This layout uses buttons.

Info page:
