Smartphone Wireframes Version 1

by Brad Simbizi

Iteration 1 Wireframes for smartphones:
Here are my 1st iteration wireframes for the various pages I'll have on the website for smartphones. I have tried to list some advantages and disadvantages for each layout.

Smartphone home page idea 1:

This homepage works if you look at its most basic function. The main issue is that it is a bit cluttered. Having all of the buttons fill up the screen can make it uncomfortable to navigate. The user might select something unintentionally because of the size and position of the buttons.

Smartphone Home page idea 2:

This layout is a bit cleaner than the 1st one. The user can swipe to the left to scroll through the options. Accidentally pressing buttons might still be an issue.

Smartphone Home page idea 3:

This layout is inspired by Natalie Young’s website on mobile devices. The user can click on the 3 bars in the top left corner that will bring up a sidebar type menu. From there, users are able to navigate to my various pages. They can also hit the arrow to close the menu.

Smartphone Blog page idea 1:

This layout is a combination of Home page layout 2 and home page layout 3. The sidebar menu is still there as the main page to page navigation.

Smartphone Essay page idea 1:

This layout continues throughout the remaining pages.

Smartphone Portfolio page idea 1:

Follows the same format.

Smartphone bio page idea 1:

On this page, I’ll have a small paragraph where I’ll have some random information about myself, as well as my intention with my website. In the space below it, I’ll provide my email or links to my social media pages.
